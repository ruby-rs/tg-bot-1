# tg-bot-1

Личный трекер жизни: гибкие ежедневные чекеры (не только да/нет — числа, заметки, интервалы времени, списки расходов), учёт веса, расходов и заправок, календарь-таблица и статистика.

Есть **два интерфейса поверх одной SQLite-базы**:

- **Веб-интерфейс** (FastAPI, адаптивный под ПК и телефон) — основной способ вести учёт.
- **Telegram-бот** — минимальный дополнительный канал: да/нет-чекеры за сегодня и вес.

## Развёртывание на сервере (Docker)

Полная инструкция, включая снос старого бота, — в [DEPLOY.md](DEPLOY.md). Кратко:

```bash
cp .env.example .env   # заполни WEB_PASSWORD, WEB_SECRET_KEY, BOT_TOKEN
docker compose up -d --build
```

Сайт: `http://<сервер>:8000` (вход по `WEB_PASSWORD`). Бот и веб делят данные;
чтобы они вели одного пользователя, укажи свой Telegram id в `WEB_USER_TG_ID`.

## Локальный запуск (для разработки)

```bash
# веб
pip install -r requirements-web.txt
DB_PATH=life_tracker.db WEB_PASSWORD=test WEB_SECRET_KEY=dev uvicorn webapp.app:app --reload

# бот
pip install -r requirements.txt
cp .env.example .env   # укажи BOT_TOKEN
python -m bot.main
```

## Веб-интерфейс

Адаптивная раскладка: на телефоне — нижняя навигация из 4 пунктов (Сегодня / Календарь / Расходы / Ещё), на широком экране — боковое меню.

### Сегодня

Чекеры за сегодня. Навигация ‹ Сегодня 📅 › позволяет листать дни назад/вперёд без захода в календарь. Шесть типов пунктов:

- **Да/нет** — тап переключает ➖ → ✅ → ❌ → снова ➖.
- **Число** — значение за день с единицей и опциональной целью (например, «Вода», литры).
- **Заметка** — список из нескольких текстовых записей за день.
- **Интервал времени** — время «с–до», можно добавлять несколько за день (сводится в часы за неделю/месяц); опционально время суток, подтип (список или свободный текст) и сумма. Используется для строек, смен, подработки, тренировок, встреч, мероприятий.
- **Список расходов** — наименование + цена, можно добавить несколько записей за день; уходит в общую статистику расходов (Покупки, Подарок, Обслуживание авто).
- **Заправка** — АЗС, литры, сумма, способ оплаты; тоже попадает в общие расходы.

Ипотека ограничена 16-м числом месяца — кнопка неактивна в другие дни.

### Календарь

Мини-календарь месяца (тап по дню открывает его на странице «Сегодня») + полная таблица-сетка «чекер × день» с итоговым столбцом по каждому типу (дни/часы/сумма) — как в исходном эксель-трекере, с горизонтальной прокруткой на телефоне.

### Чекеры (конструктор)

«Ещё» → «Чекеры»: создание/редактирование/удаление своих пунктов любого из шести типов, с настройкой полей (единица и цель для чисел; период дня, подтип из списка или свободный текст, показ суммы — для интервалов; категория расходов — для списков расходов/заправки).

### События

«Ещё» → «События»: годовщины и особые даты. При создании выбираешь режим — разовая дата со счётом вперёд («вместе уже N дней») или ежегодная с обратным отсчётом до следующей даты («до дня рождения N дней»). Ближайшие показываются чипами на панели.

### Расходы

Разбивка по категориям (сумма, доля от общих трат, полоса прогресса) с фильтром по категории; ручное добавление расхода спрятано в раскрывающийся блок; список операций с удалением.

### Вес / Статистика

Вес — запись, история, удаление. Статистика — итоги месяца по каждому чекеру (для да/нет — дней из месяца, для чисел — сумма, для интервалов — часы и сумма, для списков расходов/заправки — сумма, для заметок — число записей).

## Telegram-бот

Минимальный: панель с да/нет-чекерами за сегодня (тап переключает статус) и быстрый ввод веса. Постоянное сообщение-панель редактируется на месте, id хранится в БД (переживает рестарт бота). Всё остальное — задачи, календарь, расходы, статистика, конструктор чекеров, события — только в веб-интерфейсе.

- `/panel` — панель с да/нет-чекерами за сегодня
- `/weight <кг>` — записать вес (или без аргумента — спросит)
- `/clear` — удалить последние сообщения в чате и показать свежую панель (в пределах 48 часов — ограничение Telegram)
- `/cancel` — отменить текущее действие
- `/help` — справка

Данные хранятся в SQLite (`life_tracker.db` по умолчанию, путь настраивается через `DB_PATH`).
