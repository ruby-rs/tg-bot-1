# tg-bot-1

Личный трекер жизни: ежедневные чекеры по категориям (спорт, стройка, работа, финансы, авто и т.д.), учёт веса и расходов (включая заправки), календарь и статистика.

Есть **два интерфейса поверх одной SQLite-базы**:

- **Веб-интерфейс** (FastAPI, мобильный-first) — основной способ вести учёт с телефона через браузер.
- **Telegram-бот** — опциональный канал для быстрого ввода/просмотра.

## Развёртывание на сервере (Docker)

Полная инструкция, включая снос старого бота, — в [DEPLOY.md](DEPLOY.md). Кратко:

```bash
cp .env.example .env   # заполни WEB_PASSWORD, WEB_SECRET_KEY, BOT_TOKEN
docker compose up -d --build
```

Сайт: `http://<сервер>:8000` (вход по `WEB_PASSWORD`). Бот и веб делят данные;
чтобы они вели одного пользователя, укажи свой Telegram id в `WEB_USER_TG_ID`.

## Локальный запуск (для разработки)

```bash
# веб
pip install -r requirements-web.txt
DB_PATH=life_tracker.db WEB_PASSWORD=test WEB_SECRET_KEY=dev uvicorn webapp.app:app --reload

# бот
pip install -r requirements.txt
cp .env.example .env   # укажи BOT_TOKEN
python -m bot.main
```

## Веб-интерфейс

Адаптивный интерфейс: на телефоне — нижняя навигация из 4 пунктов (Сегодня / Календарь / Расходы / Ещё), на широком экране — боковое меню и более просторная раскладка.

- **Сегодня** — чекеры за сегодня + ближайшие годовщины/события; сводка по весу и расходам.
- **Календарь** — мини-календарь месяца (тап по дню открывает его) + полная таблица-сетка «чекер × день» за месяц с итоговым столбцом (как в эксель-трекере), с горизонтальной прокруткой на мобильном.
- **Чекеры** («Ещё» → Чекеры) — создание/редактирование/удаление своих пунктов трёх типов:
  - **Да/нет** — тап переключает ➖ → ✅ → ❌ → снова ➖ (как раньше).
  - **Число** — ввод значения за день, с опциональной единицей (г, л, ч…) и целью.
  - **Заметка** — список из нескольких текстовых записей за день (например, «Интерактивы»: куда ходили, что делали).
- **События** («Ещё» → События) — годовщины и особые даты двух типов: разовая дата со счётом вперёд («вместе уже N дней») или ежегодная с обратным отсчётом до следующей даты («до дня рождения N дней»).
- **Вес** — запись и история веса.
- **Расходы** — добавление расхода (для «Машины» — поля заправки: АЗС, литры, оплата), список и итог за месяц.
- **Статистика** — итоги месяца по каждому чекеру (для да/нет — дней из месяца, для числа — сумма, для заметки — число записей).

## Telegram-бот

Основа бота — **панель** (`/panel`): постоянное сообщение-доска, которое всегда висит в чате и только обновляется на месте. На ней — ежедневные чекеры (пункты вроде «Вода», «Спорт», «Заправка», «Рабочая смена»), которые отмечаешь тапом: ➖ не отмечено → ✅ сделал → ❌ пропустил → снова ➖. Плюс сводка: отмечено сегодня X/N, текущий вес со спарклайном, расходы за месяц.

**Ввод данных** (расход, вес, заправка, задачи) идёт в отдельном временном сообщении, которое удаляется после завершения. Панель при этом не пропадает — она обновляется. Введённые тобой текстовые сообщения тоже удаляются, чтобы чат не засорялся.

Id сообщений панели и временного окна хранятся в БД (не в памяти процесса), поэтому рестарт бота (например, после деплоя) не приводит к появлению второй панели — старое сообщение находится и редактируется как прежде.

Бот показывает и переключает только чекеры типа «да/нет». Числовые чекеры и чекеры-заметки (созданные на сайте) в боте не отображаются — их можно вести только в веб-интерфейсе.

## Команды

- `/start` — регистрация + панель
- `/panel` (и `/habits`) — показать/обновить панель с чекерами за сегодня
- `/calendar` — календарь по дням: выбери месяц (◀️/▶️), затем день, чтобы отметить чекеры и записать заправку задним числом; дни с отмеченными чекерами помечены ✅
- `/expense` — записать расход (для категории «Машина» — сценарий: заправка → литры → стоимость → способ оплаты)
- `/weight <кг>` — записать вес (или без аргумента — спросит)
- `/stats` — статистика чекеров за месяц
- `/tasks` (и `/add`) — задачи (список дел) — второстепенно, спрятано за кнопкой «📋 Задачи»
- `/tips` — советы по питанию и набору массы
- `/clear` — удалить последние сообщения в чате и показать свежую панель (в пределах 48 часов — ограничение Telegram)
- `/cancel` — отменить текущее действие
- `/help` — справка

Список команд с описаниями регистрируется в Telegram при старте (`set_my_commands`), поэтому они видны в меню бота («/» рядом с полем ввода).

Данные хранятся в SQLite (`life_tracker.db` по умолчанию, путь настраивается через `DB_PATH`).
